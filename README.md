## Game rules — PopOut

PopOut is a Connect-4 variant. Players may **drop** a piece into a column
or **pop** one of their own pieces from the bottom (all pieces above shift down).
First to connect four wins.

Three special rules:

| Rule | Condition | Result |
|------|-----------|--------|
| 1 | Pop creates simultaneous 4-in-a-row for both | Popping player wins |
| 2 | Board is full | Current player may declare a draw |
| 3 | Same state repeated 3 times | Either player may declare a draw |

## Key implementation notes

- **scikit-learn** is used **only** to load the Iris dataset.
  All tree construction, prediction, and evaluation is implemented from scratch.
- MCTS rollouts support two strategies: `random` and `heuristic`
  (one-ply look-ahead for immediate wins/blocks).
- The Decision Tree supports categorical and continuous features.
  Continuous features use binary threshold splits (maximising information gain).
- The PopOut dataset is generated by MCTS self-play and cached to `data/popout_mcts.csv`.
